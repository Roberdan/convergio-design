# Migration Guide: v6.4.0 → v6.5.0

## Breaking Changes

### Maranello IIFE facade removed from ESM barrel

The `Maranello` namespace object is no longer exported from the ESM package root.

**Before (v6.4.0):**
```ts
import { Maranello } from '@convergio/design-elements';
Maranello.header.init(el, opts);
```

**After (v6.5.0):**
```ts
import { header } from '@convergio/design-elements';
header.init(el, opts);
```

The IIFE build (`dist/iife/maranello.min.js`) still provides `window.Maranello` for script-tag consumers. This change only affects ESM imports.

## New Features

### SSR-safe imports

Root barrel imports of both packages now work in Node.js without crashing. This enables proper Next.js server component support.

### registerAll() SSR behavior

`registerAll()` now returns immediately (no-op) when `window` or `customElements` are unavailable. Safe to import unconditionally; call it in a client boundary.

### Subpath exports preserved

All subpath exports from v6.4.0 are preserved in v6.5.0, including `./charts`, `./gantt`, `./gauge`, `./controls`, and `./forms`. CJS `require` conditions remain available for all subpaths. Two new knowledge subpaths were added: `./knowledge/components` and `./knowledge/templates`.

### Starter demo pages

Four new self-contained HTML demos showcase the starter templates with realistic data and all 6 themes:
- `demo/starter-workspace.html`
- `demo/starter-ops-dashboard.html`
- `demo/starter-executive-cockpit.html`
- `demo/starter-program-management.html`
